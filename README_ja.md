# Smart Img2Img Composer v1.1.2 Stable

[English Version (README.md)](README.md)

![Smart Img2Img Composer 全体概要](docs/images/1ja.png)

## 概要
AUTOMATIC1111 Stable Diffusion WebUI 向けの、img2imgワークフローを統合制御するための高度なプロンプト構築・アセット管理アーキテクチャです。
参照画像からの自律タグ解析、メモファイルとの同期、LoRAやワイルドカードの独自の抽選システムなどを備え、煩雑なimg2imgの大量処理やプロンプト構築を自動化し、作業効率を飛躍的に向上させます。

## 主要機能 (Features)

### 🧠 参照画像の自律解析とプロンプト構築エンジン (Tagger Analyzer)

![タグ解析とプロンプト自動生成](docs/images/2ja.png)

* **Prompt Generator タブ**にて、WD14 Taggerと連携し、深層学習を用いて画像から高精度なタグを抽出します。
* 構図、背景、人物、ライティングなど、細分化されたカテゴリごとにスライダーが用意されており、抽出するタグの「信頼度（Confidence）のしきい値」を個別に細かくコントロール可能です。
* 画像からモザイク属性を検知し、該当するタグを自動で付与する機能も搭載しています。
* 納得のいくプロンプトが抽出できたら、専用ボタンを押すだけで**プロンプトと画像を保持したまま自動的にimg2imgタブへ画面遷移（1クリック・ワープ）**し、すぐに生成を開始できます。

### 🔗 ファイル名駆動型のプロンプト自動同期システム

* **Reference Image Folder**（参照画像フォルダ）内のファイル名と、**Memo File**（.txt）内に記述したセクション名（例：`[image_01]`）を自動でマッチングします。
* UI上の **Match Threshold（一致度しきい値）** スライダーを設定することで、完全一致でなくても柔軟に紐付けを行い、大量の画像処理時にも「画像ごとの固有プロンプトや特定のLoRA」をシームレスに適用します。

### 🛡️ プロファイル依存型ネガティブプロンプトの自動最適化 (Smart Negative)

![スマート・ネガティブ機能](docs/images/3ja.png)

* UIの **Active Profile** から使用するモデル（SDXL、Pony、Illustriousなど）を選択し、**Smart Negative** を有効にすると、そのモデルに必須の最適なネガティブプロンプトが自動で追加・上書きされます。
* 同時にプロンプトの並び順も最適化されます。例えばPonyプロファイルを選択した場合は、`score_9, score_8_up...` といった重要な品質タグが自動的にプロンプトの最前列にソートされ、生成品質の底上げを行います。

### 📦 資産の偏りを防ぐ、自律型インベントリ制御アルゴリズム (Inventory Logic)

![インベントリ制御システム](docs/images/4ja.png)

* **Asset Lists** タブで登録したランダム用のLoRAやワイルドカードの使用回数をシステムが記憶し、**Inventory** タブのダッシュボードで現在の「選出回数」を一覧確認できます。
* ランダム生成時に、このインベントリの記録を参照し「まだ一度も使われていないアセット」や「使用回数が少ないアセット」ほど高確率で選出される独自の抽選ロジックが働きます。これにより、同じLoRAばかりが選ばれるマンネリ化を完全に防止します。

### 🪄 構文の自動修復と独自辞書によるタグ展開プロトコル

* **Custom Dictionary（カスタム辞書）** 機能に `night, city > cyberpunk cityscape, neon lights...` のように登録しておくと、簡単な短いメモ書きをリッチなプロンプトへ全自動で変換・展開します。
* **Prompt Polish（プロンプト整形）** 機能を有効にすると、手入力やメモの結合時に発生しやすい「余分なカンマの連続（`, ,`）」や空白、閉じカッコのズレなどの構文エラーを生成直前に自動でクリーンアップします。

### 🎛️ フェイルセーフ設計と高密度UIによる操作環境の洗練

![操作を快適にする洗練された設計](docs/images/5ja.png)

* **Health Check**: 設定したフォルダやテキストファイルのパスが間違っていないかをリアルタイムに検証し、UI上に ✅ / ❌ で表示するため、生成エラーを未然に防ぐことができます。
* **Compact UI**: 画面スペースを圧迫しがちなGradio標準のUI配置を独自に最適化し、スライダーやチェックボックスをコンパクトな1行に整理しています。
* **Smart Output Sorting**: img2imgタブ内の **Output Settings** から **Sort Mode** を変更するだけで、WebUIの標準保存パスを壊すことなく、「プリセット名」や「日付」ごとのサブフォルダを自動で作成し、画像を綺麗に仕分け・命名して保存させることが可能です。

## インストール (Installation)
1. AUTOMATIC1111 WebUIの `Extensions` タブを開きます。
2. `Install from URL` タブを選択し、本拡張機能のリポジトリURLを入力してインストールします。
3. WebUIを再起動（Apply and restart UI）します。

*※タグ解析機能（Auto-Prompt Gen）を使用する場合は、事前に「WD14 Tagger」拡張機能がインストール・有効化されている必要があります。*

## ワークフローと使い方 (Usage)
本システムは、独立した専用タブでの「準備・テスト」と、img2imgタブでの「実行」が連動して動作します。

### 1. ⚙️ 全般設定とプリセット管理 (Global Settings)
拡張機能タブ「Smart Img2Img Composer」の「Settings & Preview」を開きます。
* **パスの設定**: `Reference Image Folder`（参照画像）と `Memo File`（プロンプト記述ファイル）のパスを指定します。正しく認識されるとヘルスチェックが ✅ になります。
* **プロファイルと最適化**: 使用するモデルに合わせて `Active Profile` を選択し、必要に応じて `Smart Negative` や `Prompt Polish` などの機能を有効化します。
* **プリセット保存**: 設定した内容は「Preset」として保存し、ドロップダウンメニューから瞬時に環境設定を呼び出すことが可能です。

### 2. 🏷️ タグ解析と1クリック・ワープ (Prompt Generator)
「Prompt Generator」タブでは、個別の画像に対する解析とテスト生成が可能です。
* 画像をアップロードするとWD14 Taggerが解析を実行します。カテゴリごとのスライダーを調整し、出力するタグの精度を最適化します。
* 納得のいくプロンプトが構築できたら、専用ボタンを押すことで **プロンプトと画像を保持したまま自動的にimg2imgタブへワープ（画面遷移）** します。

### 3. 🎲 アセット管理と在庫状況 (Asset Lists & Inventory)
* **Asset Lists**: ランダム選出に使用するキャラクターLoRA、シチュエーションLoRA、各種ワイルドカードのリストをWebUI上から直接管理・編集します。
* **Inventory Control**: インベントリモードを有効にしている場合、このタブから各アセットの「現在の使用回数（在庫）」を可視化・確認できます。必要に応じて在庫記録のリセットが可能です。

### 4. 🖼️ img2imgでの生成実行と自動整理 (Img2Img Execution)
`img2img` タブ内の「Smart Img2Img Composer」アコーディオンを展開し、挙動を設定します。
* **Enable**: チェックを入れると、拡張機能が指定フォルダから画像を自動取得し、メモ同期・タグ抽出・インベントリ抽選・ネガティブ最適化を全て適用した上で生成処理を代行します。
* **ランダムスロット**: キャラクターやシチュエーションLoRAをプロンプトの「前」「後」または「特定タグの後」のどこに挿入するかを指定します。
* **自動整理 (Output Settings)**: 生成された画像をサブフォルダに綺麗に分けたい場合は、このアコーディオン内の `Sort Mode` を「By Preset」などに変更してください。
